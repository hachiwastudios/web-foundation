MIGRATION GUIDE (Optional Section)
==============================================

For Users Upgrading from v1.0 to v1.5:

1. Update Package:
   ```bash

   ```

2. (Optional) Update File References:
   ```html
   <!-- Old -->
   <link rel="stylesheet" href="assets/css/utility.css">
   <script src="assets/js/hachiwa.js"></script>

   <!-- New (Recommended) -->
   <link rel="stylesheet" href="assets/css/utility.min.css">
   <script src="assets/js/hachiwa.min.js"></script>
   ```

   ```html
   <!-- Old (still works) -->

     <use href="assets/icons.svg#icon-heart"></use>
   ```

FAQ ADDITIONS
==============================================

Q: Do I need to rebuild my project for v1.5?
A: No! v1.5 is 100% backward compatible. Your v1.0 code will work without changes.

Q: How do I use the new icon system?
A: Use <i data-icon="name" data-variant="solid" class="icon"></i> for the new system, or keep using SVG sprites - both work!

Q: Do I have to use the VSCode extension?
A: No, it's optional but highly recommended for better developer experience.

Q: Are the minified files required?
A: For production, we recommend using .min.css and .min.js for 35% smaller file sizes. Dev files (.css, .js) still work fine.

Q: What happened to the old icons?
A: They're all still there! We extracted them into standalone files AND kept the sprite sheet for backward compatibility.
